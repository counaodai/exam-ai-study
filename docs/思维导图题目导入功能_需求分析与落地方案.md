## 一、需求全面分析

### 1.1 现状梳理

当前系统的题目→思维导图链路如下：

```
用户提问 → AI回答 → 后台异步分类 → 创建Question记录 → update_mindmap_after_classification()
```

**核心问题：题目导入完全绑定在"AI问答"这一条路径上，用户没有任何自主导入能力。**

具体痛点：

| 痛点 | 现状 | 影响 |
|------|------|------|
| **用户自主权缺失** | 题目只能在AI问答后自动导入，用户无法决定"哪些题目进导图、哪些不进" | 用户无法筛选有价值题目，无效提问也会进入导图（虽然后续有 `is_valid` 过滤，但过滤逻辑对用户不可见） |
| **导入场景受限** | 仅支持"实时提问→实时导入"，不支持从历史对话中回溯导入 | 用户想整理之前问过的题目时，只能重新提问一遍 |
| **无法兼容外部题库** | 用户手头有其他来源的题目（PDF截图、网页复制、笔记中的题目），完全无法导入 | 系统沦为"只能问AI的封闭工具"，而非"知识管理平台" |
| **无批量操作能力** | 每次只能导入一道题，无法批量选择多道题一起导入 | 效率极低，用户刷完一套真题后无法一次性归入导图 |
| **缺少导入前编辑** | 导入即定型，用户无法在导入前调整题目内容、修改分类标签 | 分类错误需要事后修正，增加操作成本 |
| **缺少状态反馈** | 导入后仅更新 `question_count` 数字，用户不知道"刚导入了哪道题" | 用户无法确认导入结果，也无法快速定位新导入的题目 |

### 1.2 用户核心诉求拆解

#### 模块A：自主选择已生成题目导入

| 子需求 | 描述 | 优先级 |
|--------|------|--------|
| A1. 历史题目列表多选导入 | 在AI问答会话页/历史记录页，提供题目列表视图，支持多选后批量导入到思维导图 | P0 |
| A2. 单题快捷导入 | 在每条AI回答下方，增加"加入思维导图"快捷按钮，一键导入当前题目 | P0 |
| A3. 导入前分类确认 | 导入时展示AI自动分类结果，用户可修改分类后再确认导入 | P0 |
| A4. 去重校验 | 同一道题不重复导入到同一思维导图节点下 | P0 |
| A5. 跨会话批量导入 | 支持从不同对话中选取题目，统一批量导入 | P1 |
| A6. 按模块筛选导入 | 支持按一级/二级模块筛选题目，快速定位目标题目 | P1 |

#### 模块B：复制文本/链接导入外部题目

| 子需求 | 描述 | 优先级 |
|--------|------|--------|
| B1. 粘贴识别 | 支持粘贴纯文本、Markdown格式的题目内容，自动识别题目结构 | P0 |
| B2. 格式兼容 | 兼容常见题目格式：`题干 + 选项(A/B/C/D) + 答案 + 解析` | P0 |
| B3. 内容预处理 | 粘贴后自动拆分题干、选项、答案、解析等字段 | P0 |
| B4. 用户确认再导入 | 解析结果展示给用户确认，支持手动调整各字段后再导入 | P0 |
| B5. 错误提示 | 格式不兼容、内容为空、内容违规时给出明确提示 | P0 |
| B6. 多题批量粘贴 | 支持一次粘贴多道题目（用分隔符区分），批量解析和导入 | P1 |
| B7. 链接解析 | 粘贴公考题库网页链接，自动抓取题目内容（需后端支持） | P2 |

### 1.3 用户场景调研

#### 场景一：学生刷题整理（优先级最高）

> 小明正在备考2026国考，他每天在系统中向AI提问10-20道真题。一周下来积累了100+道题目，但很多题目分类不够准确。他希望在周末集中整理这些题目，将确认有价值的题目批量导入思维导图，并按正确分类归入对应节点。

**功能优先级**：模块A（自主选择导入）> 模块B（复制导入）
**关键路径**：历史题目列表 → 多选 → 确认分类 → 批量导入

#### 场景二：职场项目知识梳理（中等优先级）

> 小红是产品经理，她用这个系统管理产品知识体系。她从公司Wiki、飞书文档中复制了大量产品方法论内容，希望粘贴到系统中，自动识别结构化内容并导入思维导图。

**功能优先级**：模块B（复制导入）> 模块A
**关键路径**：粘贴文本 → 自动解析 → 手动调整 → 确认导入

#### 场景三：个人知识体系搭建（中等优先级）

> 小李是终身学习者，他用系统搭建个人知识体系。他从各种渠道（微信公众号、知乎、书籍摘抄）收集知识点，希望快速导入到对应分类节点下，逐步完善知识图谱。

**功能优先级**：模块B（复制导入）
**关键路径**：粘贴文本 → 选择分类 → 导入

### 1.4 可行性评估

#### 技术实现难点

| 难点 | 说明 | 风险等级 |
|------|------|----------|
| **题目内容解析** | 外部粘贴的题目格式千变万化，需要鲁棒的解析策略。纯规则解析覆盖率有限，复杂格式需要LLM辅助 | 高 |
| **去重逻辑** | 需要设计合理的去重策略（基于内容相似度还是精确匹配？），避免误判 | 中 |
| **批量导入性能** | 一次导入50+道题时，需要逐题分类+创建Question+更新导图，耗时可能较长 | 中 |
| **分类准确性** | 粘贴导入的题目没有AI问答上下文，分类器仅依赖题目文本，准确率可能下降 | 中 |
| **内容审核** | 需要过滤违规内容（广告、敏感信息），纯规则过滤覆盖面有限 | 低 |

| 模块B：复制导入 | 2个API + 解析引擎 + 审核 | 1个新页面/弹窗 + 1个新组件 | 约5人天 |
| 产品体验优化 | 状态推送 | 进度提示 + 跳转定位 + 编辑功能 | 约2人天 |
| 测试 | — | — | 约2人天 |
| **总计** | | | **约14人天** |

#### 对现有系统的影响

| 影响点 | 说明 | 规避方案 |
|--------|------|----------|
| `Question` 模型 | 需新增 `source` 枚举值 `"手动导入"` `"粘贴导入"` | 不修改表结构，仅扩展 source 字段的取值约定 |
| `update_mindmap_after_classification` | 当前函数每次只处理一道题，批量导入需要批量调用 | 新增 `batch_update_mindmap` 函数，复用单题更新逻辑 |
| 分类器 `classifier.py` | 粘贴导入的题目需要先分类才能导入，当前分类器依赖AI问答上下文 | 粘贴导入时显式调用分类器，分类结果作为导入前的默认值 |
| 前端 Chat.vue | 需要在消息列表增加"导入到导图"按钮，需要增加题目列表视图 | 通过组件化方式增量添加，不影响现有对话功能 |
| 前端 MindMapDetail.vue | 需要增加"导入题目"触发按钮 | 在 header-right 区域增加按钮，不影响画布交互 |

#### 风险点与规避方案

| 风险 | 概率 | 影响 | 规避方案 |
|------|------|------|----------|
| LLM解析粘贴内容不稳定 | 中 | 高 | 优先使用规则解析（正则），LLM作为兜底；解析结果始终展示给用户确认 |
| 批量导入超时 | 中 | 中 | 后端使用后台任务异步处理，前端轮询进度；单次导入上限50道 |
| 去重误判 | 低 | 中 | 使用"题目文本相似度 > 95%"作为去重依据，而非精确匹配；去重仅警告不阻断 |
| 粘贴内容包含恶意脚本 | 低 | 高 | 后端对粘贴内容做HTML标签剥离 + 敏感词过滤 |

### 2.2 技术实现方案

#### 2.2.1 后端新增API

```
# ===== 模块A：自主选择导入 =====

GET    /api/questions
  → 获取所有已生成的题目列表（支持分页、按模块筛选、按会话筛选）
```

#### 2.2.2 内容解析规则（模块B核心）

**解析策略：规则优先 + LLM兜底**

```python
# backend/app/core/ai/question_parser.py

import re
from enum import Enum

class QuestionFormat(str, Enum):
    PLAIN = "plain"
    MARKDOWN = "markdown"

# 规则1：按题号分隔多道题目
QUESTION_SEPARATOR_PATTERNS = [
    r'(?:^|\n)\s*(\d+)[\.、．)\s]+',       # "1." "2、" "3．" "4)"
    r'(?:^|\n)\s*第[一二三四五六七八九十\d]+题',  # "第一题"
]

# 规则2：识别选项行
OPTION_PATTERNS = [
    r'^[A-J][\.、．)\s]+',                   # "A." "B、" "C．"
    r'^[（(][A-J][）)]',                     # "(A)" "（B）"
]

# 规则3：识别答案行
ANSWER_PATTERNS = [
    r'(?:答案|正确选项|参考答案)[：:]\s*([A-J]+)',
    r'^答案[：:]\s*([A-J]+)',
]

# 规则4：识别解析行
EXPLANATION_PATTERNS = [
    r'(?:解析|答案解析|解题思路|分析)[：:]',
    r'^解析[：:]',
]
```

**解析流程**：

```
粘贴文本
  │
  ├── 预处理：去除HTML标签、统一换行符、去除首尾空白
  │
  ├── 多题拆分：按题号正则匹配分割为单题文本块
  │
  ├── 逐题解析（规则优先）：
  │   ├── 提取题干（第一个题号到第一个选项之间的内容）
  │   ├── 提取选项（匹配 OPTION_PATTERNS 的行）
  │   ├── 提取答案（匹配 ANSWER_PATTERNS）
  │   ├── 提取解析（匹配 EXPLANATION_PATTERNS 到文本末尾）
  │   └── 自动分类（调用 classifier.classify_question）
  │
  ├── LLM兜底（规则解析失败时）：
  │   └── 将原始文本发送给LLM，要求返回结构化JSON
  │
  └── 返回 ParsedQuestion[] + ParseError[]
```

**ParsedQuestion 数据结构**：

```python
class ParsedQuestion(BaseModel):
    content: str                              # 题干
    options: dict[str, str] | None = None     # {"A": "选项内容", ...}
    answer: str | None = None                 # 正确答案
    explanation: str | None = None            # 解析
    suggested_module_id: UUID | None = None   # AI建议的分类模块ID
    suggested_module_path: str | None = None  # AI建议的分类路径
    parse_confidence: float = 0.0             # 解析置信度
```

#### 2.2.3 去重逻辑

```python
# backend/app/core/mindmap/dedup.py

async def check_duplicate_question(
    db: AsyncSession,
    content: str,
    mindmap_id: UUID,
    threshold: float = 0.95,
) -> tuple[bool, Question | None]:
    """
    检查题目是否已存在于指定思维导图中。
    策略：精确匹配 content 字段（去除空白和标点差异后比较）
    """
    # 获取该思维导图关联的所有模块ID
    mindmap_nodes = await db.execute(
        select(MindMapNode.module_id).where(
            MindMapNode.mind_map_id == mindmap_id,
            MindMapNode.module_id.isnot(None),
        )
    )
    module_ids = [row[0] for row in mindmap_nodes.all()]

    if not module_ids:
        return False, None

    # 标准化题目文本用于比较
    normalized = normalize_text(content)

    existing = await db.execute(
        select(Question).where(
            Question.module_id.in_(module_ids),
            Question.is_valid == True,
        )
    )
    for q in existing.scalars().all():
        if normalize_text(q.content) == normalized:
            return True, q

    return False, None


def normalize_text(text: str) -> str:
    """标准化文本：去除空白字符、统一标点"""
    import re
    text = re.sub(r'\s+', '', text)
    text = text.replace('．', '.').replace('，', ',').replace('：', ':')
    return text.lower()
```

#### 2.2.4 批量导入流程

```python
# backend/app/api/mindmaps.py 新增

@router.post("/{map_id}/import-questions")
async def import_questions_to_mindmap(
    map_id: str,
    request: ImportQuestionsRequest,
    db: AsyncSession = Depends(get_db),
):
    """
    批量导入题目到思维导图
    流程：去重 → 分类确认 → 创建Question → 更新导图节点
    """
    mindmap = await db.get(MindMap, uuid.UUID(map_id))
    if not mindmap:
        raise HTTPException(status_code=404, detail="思维导图不存在")

    imported = []
    failed = []

    for item in request.questions:
        try:
            # 1. 去重检查
            is_dup, existing = await check_duplicate_question(
                db, item.content, mindmap.id
            )
            if is_dup and not request.allow_duplicate:
                failed.append({
                    "content_preview": item.content[:50],
                    "reason": "题目已存在于该思维导图中",
                    "existing_question_id": str(existing.id) if existing else None,
                })
                continue

            # 2. 确定分类（用户指定 > AI建议 > 思维导图根模块）
            module_id = item.module_id or item.suggested_module_id
            if not module_id:
                module_id = await find_module_id_by_name(db, mindmap.root_module)

            # 3. 创建Question记录
            question = Question(
                content=item.content,
                options=item.options,
                answer=item.answer,
                explanation=item.explanation,
                module_id=module_id,
                source="手动导入",
                mastery=0,
            )
            db.add(question)
            await db.flush()

            # 4. 更新思维导图
            await update_mindmap_after_classification(
                db,
                primary_module=mindmap.root_module,
                secondary_module=item.secondary_module,
                question_id=question.id,
            )

            imported.append(str(question.id))

        except Exception as e:
            logger.error(f"导入题目失败: {e}")
            failed.append({
                "content_preview": item.content[:50],
                "reason": str(e),
            })

    await db.commit()

    return {
        "imported_count": len(imported),
        "imported_ids": imported,
        "failed": failed,
    }
```

#### 2.2.5 内容审核机制

```python
# backend/app/core/ai/content_filter.py

import re

# 敏感词库（示例，实际应从配置读取）
BLOCKED_PATTERNS = [
    r'(广告|推广|加微信|扫码|免费领取)',
    r'(赌博|博彩|彩票)',
    r'(色情|淫秽)',
    # ... 更多规则
]

# 最小题目长度
MIN_QUESTION_LENGTH = 10

# 最大题目长度
MAX_QUESTION_LENGTH = 5000


def validate_question_content(content: str) -> tuple[bool, str | None]:
    """
    审核题目内容
    返回: (是否通过, 拒绝原因)
    """
    if not content or len(content.strip()) < MIN_QUESTION_LENGTH:
        return False, "题目内容过短，请检查后重试"

    if len(content) > MAX_QUESTION_LENGTH:
        return False, f"题目内容过长（超过{MAX_QUESTION_LENGTH}字符），请拆分后导入"

    for pattern in BLOCKED_PATTERNS:
        if re.search(pattern, content, re.IGNORECASE):
            return False, "内容包含不适宜信息，无法导入"

    return True, None
```
