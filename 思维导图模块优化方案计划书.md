# 思维导图模块优化方案计划书

> **项目**：公考AI智能学习系统  
> **模块**：思维导图（MindMap）  
> **目标**：补充手动编辑能力 + 节点关联题目查看功能  
> **日期**：2026-05-29

---

## 一、背景与目标

### 1.1 现状

当前思维导图模块已实现：

- ✅ vue-flow 画布渲染（缩放、平移、MiniMap、连线动画）
- ✅ 自定义节点组件（类型标签、题目数、掌握度、方法论标记）
- ✅ AI 问答后自动创建/更新导图节点
- ✅ 后端 CRUD 基础 API（列表、详情、创建、更新节点位置）

### 1.2 待解决问题

| 问题 | 影响 |
|------|------|
| 用户无法手动添加/删除/修改节点 | 导图是"只读"的，无法个性化调整 |
| 点击节点无法查看关联题目 | 导图与复习脱节，用户价值低 |
| 拖拽节点后位置不保存 | 用户调整布局后刷新丢失 |
| 无右键菜单 | 缺乏直观的操作入口 |

### 1.3 优化目标

1. **编辑能力**：支持添加、删除、修改节点，拖拽保存位置
2. **题目关联**：点击节点即可查看该模块下的所有关联题目
3. **操作体验**：右键菜单 + 双击编辑 + 拖拽保存，符合思维导图软件操作习惯

### 2.2 API 接口设计

    parent_id: str | None = None  # 新增：支持修改父节点（拖拽改变层级）

# 删除节点响应
class DeleteNodeResponse(BaseModel):
    message: str
    deleted_count: int            # 被删除的节点总数（含子节点）

# 节点关联题目
class NodeQuestionItem(BaseModel):
    id: str
    content: str
    answer: str | None
    explanation: str | None
    source: str | None
    difficulty: int | None
    mastery: int

    total: int
    page: int
    page_size: int
    items: list[NodeQuestionItem]

## 四、关键实现细节

### 4.1 删除节点的级联逻辑

```python
# 后端 mindmaps.py
async def _delete_node_recursive(db: AsyncSession, node_id: uuid.UUID) -> int:
    """递归删除节点及其所有子节点，返回删除总数"""
    children_result = await db.execute(
        select(MindMapNode).where(MindMapNode.parent_id == node_id)
    )
    children = children_result.scalars().all()
    deleted_count = 0
    for child in children:
        deleted_count += await _delete_node_recursive(db, child.id)

    node = await db.get(MindMapNode, node_id)
    if node:
        await db.delete(node)
        deleted_count += 1
    return deleted_count
```

### 4.2 关联题目查询逻辑

```python
# 通过 node.module_id 关联 questions 表
# 如果节点没有 module_id，则通过 label 匹配 module.name
async def get_node_questions(db, map_id, node_id, page, page_size):
    node = await db.get(MindMapNode, uuid.UUID(node_id))
    if not node:
        raise HTTPException(404, "节点不存在")

    # 优先使用 module_id 直接关联
    if node.module_id:
        query = select(Question).where(Question.module_id == node.module_id)
    else:
        # 通过 label 匹配 module name
        module_result = await db.execute(
            select(Module).where(Module.name == node.label)
        )
        module = module_result.scalar_one_or_none()
        if not module:
            return {"total": 0, "items": []}
        query = select(Question).where(Question.module_id == module.id)

    # 分页查询
    count_result = await db.execute(select(func.count()).select_from(query.subquery()))
    total = count_result.scalar()

    items_result = await db.execute(
        query.order_by(Question.created_at.desc())
        .offset((page - 1) * page_size)
        .limit(page_size)
    )
    return {"total": total, "items": items_result.scalars().all()}
```

## 六、风险与注意事项

| 风险 | 应对措施 |
|------|----------|
| 删除节点误操作 | 添加 `el-popconfirm` 二次确认，显示将被删除的子节点数量 |
| 拖拽保存高频请求 | 使用 500ms 防抖，仅在拖拽停止时触发 |
| module 类型节点被误删 | 前端菜单禁用 + 后端接口校验，module 类型不允许删除 |
| 关联题目查询性能 | 使用分页查询，默认每页 10 条，避免一次性加载大量数据 |
| vue-flow 拖拽改变层级 | P2 优先级，本期仅保存位置，不改变 parent_id |

## 七、验收标准

| 编号 | 验收项 | 验收标准 |
|------|--------|----------|
| AC-01 | 添加节点 | 右键节点 → 添加子节点 → 输入名称 → 节点出现在导图中，刷新后仍在 |
| AC-02 | 删除节点 | 右键 topic 节点 → 删除 → 确认 → 节点及其子节点消失，数据库已删除 |
| AC-03 | 编辑节点 | 双击节点 → 修改名称 → Enter 保存 → 名称更新，刷新后仍为新名称 |
| AC-04 | 拖拽保存 | 拖拽节点到新位置 → 刷新页面 → 节点位置保持不变 |
| AC-05 | 查看关联题目 | 点击节点 → 右侧弹出 Drawer → 显示该模块关联题目列表（分页） |
| AC-06 | 题目解析展开 | 点击题目 → 展开显示答案和解析内容 |
| AC-07 | 右键菜单 | 右键节点 → 弹出菜单，module 类型无删除选项 |
| AC-08 | 根节点保护 | 右键根节点（module 类型） → 删除按钮禁用 |
