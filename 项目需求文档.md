# 公考AI智能学习系统 — 项目需求说明书

> **版本**: v2.0
> **日期**: 2026-05-22
> **技术栈**: Vue3 + Element Plus + Python FastAPI + LangChain + ChromaDB
> **目标**: 构建一个基于 RAG 知识库的公考智能学习助手，集成 AI 问答、思维导图自动生成、题目分类统计和通用解题方法归纳四大核心能力。

---

## 一、项目概述

### 1.1 项目背景

公务员考试（公考）备考过程中，考生面临以下痛点：

- 资料繁多（教材、真题、笔记），缺乏结构化整理工具
- 做过的题目容易遗忘，同类题型反复出错
- 缺少将零散知识点系统化为"知识体系"的手段
- 无法从大量做题经验中提炼通用解题方法

### 1.2 项目目标

打造一个 **"资料导入 → RAG知识库 → AI问答 → 自动归类 → 思维导图 → 方法论提炼"** 的完整学习闭环系统。

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 知识结构化 | 将零散资料自动组织为可检索的知识库 |
| 学习可视化 | 思维导图直观展示知识体系和掌握程度 |
| 智能问答 | 基于知识库的 RAG 增强问答，答案可溯源 |
| 自动积累 | 每次提问自动归类，知识体系越用越完善 |
| 方法论提炼 | 积累足够题目后，AI 自动总结同类题型通用解法 |

## 二、技术栈定义

### 2.1 整体技术方案

> **架构原则**：前后端分离，前端 Vue3 负责 UI 交互，后端 Python FastAPI 负责 RAG/AI 核心逻辑。两者通过 REST API 通信。

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端框架** | **Vue 3** (Composition API + `<script setup>`) | 熟悉度高，开发效率最高 |
| **UI组件库** | **Element Plus** | 丰富的后台管理组件，表格/表单/上传/对话框开箱即用 |
| **思维导图渲染** | **vue-flow** | react-flow 的 Vue 版本，同一作者，API 一致，支持自定义节点、拖拽、缩放、连线 |
| **图表** | **ECharts** (vue-echarts) | Vue 生态最常用的图表库，支持柱状图/饼图/折线图/热力图 |
| **状态管理** | **Pinia** | Vue 3 官方推荐状态管理，TypeScript 友好 |
| **HTTP客户端** | **Axios** | 请求拦截、响应拦截、错误处理完善 |
| **后端框架** | **Python FastAPI** | 异步高性能，自带 OpenAPI 文档，类型提示友好 |
| **RAG框架** | **LangChain (Python)** | Python 版是 LangChain 一等公民，文档最全、示例最多、更新最快 |
| **向量数据库** | **ChromaDB** | 本地零配置部署，Python 集成优秀，适合开发阶段；生产可切换 Milvus/Pinecone |
| **关系数据库** | **MySQL** | 存储题目、模块、思维导图结构等结构化数据 |
| **ORM** | **SQLAlchemy** + **Alembic** | Python 生态最成熟的 ORM + 数据库迁移工具 |
| **AI/LLM** | **mimo 大模型**（兼容 Anthropic 接口协议） | 问答、分类、方法论生成，通过 langchain-anthropic 调用 |
| **Embedding模型** | **BGE-M3** (推荐，中文效果好且免费) 或 **OpenAI text-embedding-3-small** | 文本向量化 |
| **文件存储** | **本地文件系统** 或 **Supabase Storage** | 存储上传的原始文档 |

### 2.2 选型理由

| 决策 | 理由 |
|------|------|
| **Vue3 而非 React/Next.js** | 开发者熟悉 Vue3 + Element Plus，开发效率高 3-5 倍；本项目不需要 SSR（无需SEO），Next.js 的核心优势无法发挥 |
| **Python FastAPI 而非 Node.js** | LangChain Python 版是主版本，RAG 相关代码示例 90% 为 Python；文档解析（PDF/Word）Python 库最成熟；遇到问题时社区资源最丰富 |
| **vue-flow 而非手绘** | react-flow 的 Vue 官方版本，API 一致，支持自定义节点、自动布局、拖拽交互，省去大量 Canvas 开发工作 |
| **前后端分离** | 前端和后端独立开发、独立部署，互不影响；后端可独立扩展（如 GPU 推理） |

### 2.3 开发环境

| 工具 | 版本要求 | 说明 |
|------|---------|------|
| **Node.js** | >= 18.0 | 前端运行环境 |
| **pnpm** | >= 8.0 | 前端包管理器（推荐） |
| **Python** | >= 3.11 | 后端运行环境 |

#### F1.1 文档上传

**详细需求**:

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|

#### F1.2 文档解析与分块

**功能描述**: 将上传的文档解析为纯文本，并按智能策略分割为知识片段。

**分块策略（核心）**:

```
分块优先级策略（从高到低）：
1. 按题目结构分块：识别题号标记（如 "1."、"一、"），每道题为一个独立 chunk
2. 按章节标题分块：识别章节标题（如 "第一章"、"## 标题"），按章节切分
3. 按段落分块：遇到连续空行时切分
4. 固定长度兜底：以上策略都不适用时，按 500 字符/块 切分，重叠 100 字符
```

**每个 Chunk 的元数据结构**:

```python
# backend/app/schemas/document.py
from pydantic import BaseModel
from typing import Optional
from enum import Enum

class ChunkType(str, Enum):
    QUESTION = "question"
    KNOWLEDGE = "knowledge"
    METHOD = "method"

class ChunkMetadata(BaseModel):
    page: Optional[int] = None               # 原文页码
    chapter: Optional[str] = None            # 所属章节
    question_number: Optional[int] = None    # 题号（如为题目）
    year: Optional[str] = None               # 年份（如为真题）
    tags: list[str] = []                     # 标签

class DocumentChunk(BaseModel):
    id: str                                  # 唯一标识
    content: str                             # 文本内容
    source: str                              # 来源文件名
    module: Optional[str] = None             # 归属模块（如有）
    chunk_type: ChunkType                    # 片段类型
    metadata: ChunkMetadata
```

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| F1.2.1 | PDF解析 | P0 | 提取文字内容，保留基本结构 |
| F1.2.2 | Word解析 | P0 | 解析.docx格式 |
| F1.2.3 | 智能分块 | P0 | 按题目/章节结构智能切分，而非固定字符数 |
| F1.2.4 | 向量化存储 | P0 | 将每个chunk向量化后存入向量数据库 |

#### F1.3 RAG 检索与问答

**功能描述**: 用户提出问题时，系统从知识库中检索相关片段，结合 LLM 生成回答。

**RAG 流程**:

```
用户提问
  │
  ▼
┌──────────────────────┐
│ 1. 查询预处理         │  对用户问题进行改写/扩展，提高检索召回率
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ 2. 向量相似度检索     │  从ChromaDB中检索 Top-K 相似片段（K=5~10）
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ 3. 重排序(Rerank)     │  对检索结果进行相关性重排序，保留 Top-N（N=3~5）
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│ 4. Prompt 构建        │  将检索到的片段作为上下文，构建完整 Prompt
└──────────┬───────────┘
           │
           ▼
┌───���──────────────────┐
│ 5. LLM 生成回答       │  基于上下文生成回答，并标注引用来源
└──────────────────────┘
```

**Prompt 模板设计**:

```
你是一位资深公考培训专家。请基于以下参考资料回答用户的问题。

## 参考资料
{retrieved_chunks}   // 检索到的知识片段，带来源标记

## 回答要求
1. 如果参考资料中有相关内容，请基于资料回答，并标注来源
2. 如果参考资料中没有直接答案，请基于你的知识回答，并明确说明"此回答非来自知识库"
3. 如果是选择题，请给出答案并详细分析每个选项
4. 使用 Markdown 格式输出

## 用户问题
{user_question}
```

| 编号 | 需求 | 优先级 | 说明 |
|------|------|--------|------|
| F1.3.1 | 对话式问答界面 | P0 | 类似ChatGPT的对话界面，支持多轮对话 |
| F1.3.2 | 引用来源展示 | P0 | 回答中标注知识来源（文件名、页码） |
| F1.3.3 | 相关片段展示 | P1 | 侧边栏展示本次检索到的相关知识片段 |
| F1.3.4 | 对话历史保存 | P1 | 保存所有对话记录，支持查看和删除 |
| F1.3.5 | 知识库选择 | P1 | 可选择在哪些已导入的知识库范围内检索 |

### 模块三：思维导图

#### F3.1 思维导图数据结构

```python
# backend/app/schemas/mindmap.py
from pydantic import BaseModel
from typing import Optional
from enum import Enum
from datetime import datetime

class NodeType(str, Enum):
    MODULE = "module"
    TOPIC = "topic"
    METHOD = "method"
    QUESTION = "question"

class NodePosition(BaseModel):
    x: float = 0.0
    y: float = 0.0

class NodeMetadata(BaseModel):
    is_auto_generated: bool = False            # 是否AI自动生成
    source_questions: list[str] = []           # 来源题目ID列表
    method_summary: Optional[str] = None       # 方法论总结（如有）

class MindMapNode(BaseModel):
    id: str
    label: str                                 # 节点显示名称
    type: NodeType
    parent_id: Optional[str] = None            # 父节点ID（树状结构）
    level: int                                 # 层级深度
    question_count: int = 0                    # 关联题目数量
    mastery: int = 0                           # 掌握度 0-100
    content: Optional[str] = None              # 节点详细内容（Markdown）
    metadata: NodeMetadata = NodeMetadata()
    position: NodePosition = NodePosition()    # 可视化位置

class MindMap(BaseModel):
    id: str
    title: str
    root_module: str                           # 一级模块名
    nodes: list[MindMapNode]
    edges: list[dict]                          # 节点间连线
    created_at: datetime
    updated_at: datetime
```

### 模块四：通用解题方法归纳

#### F4.1 功能描述

当同一子分类下积累了足够数量的题目（阈值默认为 5 道），系统自动触发 AI 分析，归纳出该类题型的通用解题方法。

#### F4.2 方法论生成 Prompt

## 五、数据库设计

### 5.1 关系数据库（MySQL）

```sql
-- 文档表
CREATE TABLE documents (
  id          CHAR(36) PRIMARY KEY,
  filename    VARCHAR(255) NOT NULL,
  file_type   VARCHAR(20) NOT NULL,          -- pdf/docx/txt/md
  file_size   INTEGER NOT NULL,
  file_path   TEXT NOT NULL,                  -- 存储路径
  module      VARCHAR(50),                    -- 指定的归属模块
  status      VARCHAR(20) DEFAULT 'pending',  -- pending/processing/completed/failed
  chunk_count INTEGER DEFAULT 0,              -- 分块数量
  created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 模块分类表
CREATE TABLE modules (
  id          CHAR(36) PRIMARY KEY,
  name        VARCHAR(100) NOT NULL,
  parent_id   CHAR(36),
  level       TINYINT NOT NULL,              -- 1=一级模块, 2=二级, 3=三级
  description TEXT,
  sort_order  INTEGER DEFAULT 0,
  is_preset   BOOLEAN DEFAULT FALSE,          -- 是否系统预置
  created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (parent_id) REFERENCES modules(id)
);

-- 题目表
CREATE TABLE questions (
  id            CHAR(36) PRIMARY KEY,
  content       TEXT NOT NULL,                  -- 题目内容
  options       JSON,                           -- 选项（选择题）
  answer        TEXT,                           -- 正确答案
  explanation   TEXT,                           -- 解析
  module_id     CHAR(36),                       -- 所属模块
  source        VARCHAR(255),                   -- 来源（文档名/手动输入）
  source_doc_id CHAR(36),                       -- 来源文档ID
  source_page   INTEGER,                        -- 来源页码
  difficulty    TINYINT,                        -- 难度 1-5
  mastery       TINYINT DEFAULT 0,              -- 掌握度 0-100
  tags          JSON DEFAULT ('[]'),
  created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (module_id) REFERENCES modules(id),
  FOREIGN KEY (source_doc_id) REFERENCES documents(id)
);

-- AI对话表
CREATE TABLE conversations (
  id          CHAR(36) PRIMARY KEY,
  title       VARCHAR(200),
  created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 对话消息表
CREATE TABLE messages (
  id              CHAR(36) PRIMARY KEY,
  conversation_id CHAR(36),
  role            VARCHAR(20) NOT NULL,         -- user/assistant/system
  content         TEXT NOT NULL,
  question_id     CHAR(36),                     -- 如果消息关联了题目
  module_id       CHAR(36),                     -- 自动分类结果
  classification  JSON,                         -- 分类详情 {module, sub_module, confidence}
  sources         JSON,                         -- RAG检索来源列表
  created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (conversation_id) REFERENCES conversations(id) ON DELETE CASCADE,
  FOREIGN KEY (question_id) REFERENCES questions(id),
  FOREIGN KEY (module_id) REFERENCES modules(id)
);

-- 思维导图表
CREATE TABLE mind_maps (
  id          CHAR(36) PRIMARY KEY,
  title       VARCHAR(200) NOT NULL,
  root_module VARCHAR(100) NOT NULL,           -- 一级模块名
  created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 思维导图节点表
CREATE TABLE mind_map_nodes (
  id              CHAR(36) PRIMARY KEY,
  mind_map_id     CHAR(36),
  parent_id       CHAR(36),
  label           VARCHAR(200) NOT NULL,
  type            VARCHAR(20) NOT NULL,         -- module/topic/method/question
  module_id       CHAR(36),
  question_count  INTEGER DEFAULT 0,
  mastery         TINYINT DEFAULT 0,
  content         TEXT,                         -- 详细内容（Markdown）
  method_summary  TEXT,                         -- 方法论总结
  is_auto_generated BOOLEAN DEFAULT FALSE,
  position_x      FLOAT DEFAULT 0,
  position_y      FLOAT DEFAULT 0,
  sort_order      INTEGER DEFAULT 0,
  created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (mind_map_id) REFERENCES mind_maps(id) ON DELETE CASCADE,
  FOREIGN KEY (parent_id) REFERENCES mind_map_nodes(id),
  FOREIGN KEY (module_id) REFERENCES modules(id)
);

-- 方法论表
CREATE TABLE method_summaries (
  id              CHAR(36) PRIMARY KEY,
  module_id       CHAR(36),
  method_name     VARCHAR(200) NOT NULL,
  recognition     JSON,                        -- 题型识别特征
  steps           JSON NOT NULL,               -- 解题步骤 [{step, title, description, example}]
  traps           JSON,                        -- 常见陷阱 [{trap, solution}]
  quick_tips      JSON,                        -- 速解技巧
  key_formulas    JSON,                        -- 关键公式
  summary         TEXT,                         -- 一句话总结
  question_count  INTEGER,                      -- 基于多少道题生成
  source_question_ids JSON,                    -- 来源题目ID列表
  is_auto_generated BOOLEAN DEFAULT TRUE,
  created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (module_id) REFERENCES modules(id)
);
```
